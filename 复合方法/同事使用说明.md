# 复合方法翻译工具 - 同事使用说明

## 需要打包的文件

给同事使用时，只需要这些文件：

- `一键启动.bat`
- `hybrid_markdown_run_translator.py`
- `requirements.txt`
- `同事使用说明.md`

不要打包这些本机文件：

- `.venv/`：本机会自动创建，体积很大。
- `.hybrid_config.json`：可能保存你的 API Key 或本机配置。
- `输出/`：这是你的历史输出结果。
- `__pycache__/`：Python 缓存文件，没有用。

## 使用方法

1. 解压整个文件夹。
2. 双击 `一键启动.bat`。
3. 第一次启动会自动创建环境并安装依赖，需要联网，可能等待几分钟。
4. 在界面里选择 API 类型，输入 API Key。
5. 选择或拖入 `.docx` 合同文件。
6. 选择输出目录。
7. 点击“开始复合方法翻译”。

## 输出文件说明

输出目录里最重要的是：

- `*_复合方法英文翻译.docx`

这个是最终英文译文 Word 文件。

输出目录里还会有一个 `过程文件` 文件夹，里面通常包括：

- `*_source_blocks.md`
- `*_translated_blocks.md`
- `*_复合方法checklist.md`
- `*_复合方法明细.json`
- `说明_这些过程文件可删除.txt`

这些文件只用于排查问题、追踪翻译过程或给开发者定位 bug。正常交付通常不需要它们。如果最终 Word 译文确认没有问题，`过程文件` 文件夹可以删除。

## 注意事项

- 电脑需要能运行 Python 3。如果双击后提示找不到 Python，请先安装 Python 3。
- API Key 不会随压缩包提供，每个人需要输入自己的 Key。
- 如果中途点击中止，程序会导出最近已经完成的内容，不再等待当前 API 批次返回。
- 目前公司名不会强制统一改成“英文名（中文名）”，以避免复核阶段反复重翻。
