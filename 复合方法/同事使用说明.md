# 复合方法翻译工具 - 同事使用说明

## 需要打包的文件

给同事使用时，只需要这些文件：

- `一键启动.bat`
- `hybrid_markdown_run_translator.py`
- `requirements.txt`
- `同事使用说明.md`

不要打包这些本机文件：

- `.venv/`：本机会自动创建，体积很大。
- `.hybrid_config.json`：可能保存你的 API Key 或本机配置。
- `输出/`：这是你的历史输出结果。
- `__pycache__/`：Python 缓存文件，没有用。

## 使用方法

1. 解压整个文件夹。
2. 双击 `一键启动.bat`。
3. 第一次启动会自动创建环境并安装依赖，需要联网，可能等待几分钟。
4. 如果电脑没有 Python，启动脚本会先尝试用 Windows 自带的 `winget` 自动安装 Python 3.12。
5. 如果自动安装失败，请手动安装 Python 3.12 或更新版本，并在安装时勾选 `Add python.exe to PATH`，然后重新双击 `一键启动.bat`。
6. 在界面里选择 API 类型，输入 API Key。
7. 选择或拖入 `.docx` 合同文件。
8. 选择输出目录。
9. 点击“开始复合方法翻译”。
10. 如果电脑屏幕较小，界面右侧有滚动条，也可以用鼠标滚轮上下滚动。

## 输出文件说明

输出目录里最重要的是：

- `*_复合方法英文翻译.docx`

这个是最终英文译文 Word 文件。

输出目录里还会有一个 `过程文件` 文件夹，里面通常包括：

- `*_source_blocks.md`
- `*_translated_blocks.md`
- `*_复合方法checklist.md`
- `*_复合方法明细.json`
- `说明_这些过程文件可删除.txt`

这些文件只用于排查问题、追踪翻译过程或给开发者定位 bug。正常交付通常不需要它们。如果最终 Word 译文确认没有问题，`过程文件` 文件夹可以删除。

## 注意事项

- 电脑需要能运行 Python 3。如果脚本无法自动安装 Python，请从 https://www.python.org/downloads/ 安装，并勾选 `Add python.exe to PATH`。
- API Key 不会随压缩包提供，每个人需要输入自己的 Key。
- 如果中途点击中止，程序会导出最近已经完成的内容，不再等待当前 API 批次返回。
- 如果自动安装 Python 时看到 `msstore` 报错，可以直接关闭窗口再双击一次；v1.15 的启动脚本会优先使用 `winget` 源安装 Python 3.12。
- v1.14 起最终英文版按“无中文残留”处理：公司名、人名、地址、目录、表格、页眉页脚里如果还有中文，都会在导出前再扫一次并尝试重翻。
- 公司名不会强制统一改成“英文名（中文名）”；现在默认使用纯英文/拼音化/官方英文名，避免在英文交付版里保留中文。
