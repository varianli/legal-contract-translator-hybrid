# 新方案：大段翻译 + 格式映射 + Checklist

这个方案用于在更大上下文中翻译法律合同，同时尽量把原文中的粗体、下划线、高亮映射到英文译文的对应短语上。

## 核心流程

1. 程序读取 Word `.docx`。
2. 程序确定性提取有格式的原文片段，例如粗体、下划线、高亮。
3. 大模型按大段上下文翻译整批段落，保证法律英语自然、术语一致。
4. 大模型为每个有格式的原文片段返回英文对应短语。
5. 程序在英文译文中定位这些短语，并重新加上原格式。
6. 程序输出 checklist，标记每个格式片段是否成功映射。

## 内置提示词逻辑

程序会明确要求模型先用大段上下文完成法律英语翻译，但格式检查必须以“每一段”为单位逐段完成：

1. 把每个段落视为独立的格式应用单位。
2. 读取该段落由 Word run 结构提取出来的 `format_spans`。
3. 把这些 span 当作 checklist：记录原文中哪些内容有粗体、下划线、高亮等格式。
4. 在该段英文译文中定位法律含义对应的英文短语。
5. 返回精确的英文 `target_text`，由程序把格式加回去。
6. 当前段落 checklist 完成后，才进入下一段。

## 输出文件

默认输出在 `新方案/输出`：

- `*_新方案英文翻译.docx`：英文译文
- `*_格式映射checklist.md`：人工检查清单
- `*_格式映射明细.json`：完整机器可读明细

## 使用方法

1. 双击 `一键启动.bat`。
2. 选择 `OpenAI` 或 `DeepSeek`。
3. 输入 API Key。
4. 选择或拖入 `.docx` 合同。
5. 确认输出目录。
6. 点击“开始新方案翻译”。
7. 如需提前停止，点击“中止并导出当前进度”。程序会等当前 API 批次返回后立刻导出已完成内容，未完成段落保留原文。

## Checklist 状态含义

- `APPLIED`：该格式片段已经在英文译文中找到对应短语并加回格式。
- `MISSING_MAPPING`：模型没有给出这个格式片段的英文映射。
- `TARGET_NOT_FOUND`：模型给了映射，但程序在英文译文中没有精确找到该短语。
- `HAS_CHINESE_IN_TRANSLATION`：译文中仍检测到中文，需要人工复核。
- `MISSING_TRANSLATION`：模型没有返回该段翻译。
- `CANCELLED`：用户主动中止，文件已按当前进度导出。

## 重要取舍

该方案不会强行保持原 Word run 的切分，而是让英文译文更自然，然后把格式映射到英文对应短语。它适合“翻译质量和上下文优先，同时需要格式审计”的场景。

如果要求 Word 内部 run 位置完全不变，请继续使用上一层目录里的“并发 run 保格式模式”。
