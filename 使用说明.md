# 法律合同中译英工具使用说明

这个文件夹里的工具面向不会编程的用户：选择 API 类型，输入 API Key，拖入 Word 合同，选择输出位置，然后导出英文译文。

## 第一次使用

1. 双击 `一键启动.bat`。
2. 第一次启动会自动创建运行环境并安装依赖，可能需要几分钟。
3. 弹出窗口后，在 `API 类型` 中选择 `OpenAI` 或 `DeepSeek`。
4. 在 API Key 输入框里粘贴你的 Key。
5. `翻译模式` 默认选择 `并发 run 保格式模式（推荐）`。
6. 把 `.docx` 合同拖进窗口，或点“选择文件”。
7. 工具会自动填一个输出文件路径；也可以点击“另存为”选择导出位置。
8. 点“开始翻译并导出”。

## 翻译模式怎么选

### 并发 run 保格式模式（推荐）

这是当前默认模式。工具会保留 Word 原来的每一个 run，只替换每个 run 里的文字。粗体、下划线、高亮、字号、颜色等 run 级格式会留在原来的 run 上。为了提速，工具会同时翻译多个段落。

优点：

- 格式保留优先级最高。
- 不依赖模型完整保留几百个标签，因此比旧的批量标签模式更稳。
- 多个段落并发翻译，比单线程逐段翻译快。
- 如果某个 run 漏翻，会只补译这个 run，不会改写整段格式。

缺点：

- 英文语序和中文不完全一致时，格式仍固定在原 run 位置，这是“格式必须完全一样”的必然取舍。
- 比不保留内联格式的整段翻译慢，但更符合合同格式要求。

### 快速批量标签模式（不推荐大文档）

这是方案二。工具会把多个段落转换成带 Word run 标签的批量文本，例如 `<r id="0">...</r>`，一次发给 API 翻译，再按标签还原回 Word 文档。

优点：

- API 调用次数少，速度快。
- 仍会尽量保留粗体、下划线、高亮等 run 级格式。
- 适合小文档或 run 数量不多的文档。

缺点：

- 如果模型大量漏掉标签，会触发补译，速度会变慢。
- 批量 API 调用期间，进度会停在当前批次；批次返回后会快速更新多个段落。

### 逐段上下文模式（更稳但慢）

这是旧模式。每个段落单独翻译，前后文更细，但 API 调用次数多，速度慢。

适合：

- 格式特别复杂的合同。
- 小文件。
- 快速批量标签模式输出不理想时的重试。

## 如何看实时进度

开始翻译后，窗口会显示：

- 已完成段落数 / 总段落数
- 百分比进度条
- 当前正在翻译的段落片段
- 下方日志会显示每段翻译和保存进度

长合同调用 API 时，某一段或某一批很长时可能会停留几十秒，这是正常的。只要日志或当前段落仍在变化，就说明工具还在工作。

在并发 run 保格式模式下，进度会按完成的段落跳动，不一定严格按原文顺序增长，这是并发导致的正常现象。

## 如何保存当前进度

默认会勾选“自动把已翻译进度保存到输出文件”。这表示每翻译完一段，工具都会把当前半成品写到你选择的输出 `.docx`。

也可以点击“保存当前进度”。按钮点击后不会打断正在进行的 API 调用，而是会在当前段落翻译完成后立刻保存。

注意：如果你在 Word 里打开了输出文件，工具可能无法覆盖保存。新版工具遇到这种情况会自动另存为带时间戳的新文件，例如 `合同_英文翻译_自动另存_20260518_153000.docx`，并在日志里提示。翻译过程中最好不要打开输出文件；需要查看时，可以先复制一份再打开。

## DeepSeek 怎么填

选择 `DeepSeek` 后，工具会自动填入：

- Base URL：`https://api.deepseek.com`
- 默认模型：`deepseek-v4-flash`

现在默认只使用 `deepseek-v4-flash`，速度更快，也更适合大文档批量处理。

## OpenAI 怎么填

选择 `OpenAI` 后：

- Base URL 可以留空。
- 默认模型是 `gpt-4.1`。
- 如果你有其他可用模型，也可以直接改模型名称。

## 支持什么文件

当前版本只支持 `.docx`。

推荐流程：

- Word/WPS 合同：先另存为 `.docx`，再拖入工具。
- PDF 合同：先用 Word、Adobe Acrobat 或 OCR 工具转换成 `.docx`，检查排版后再翻译。
- 扫描件图片：先 OCR 成可编辑 Word 文档，再翻译。

## 能保留哪些格式

工具会尽量保留 Word 文档里的：

- 段落样式
- 表格结构
- 页眉和页脚
- 粗体
- 下划线
- 高亮
- 颜色、字号等 run 级格式

注意：文本框、复杂形状、嵌入对象、批注、修订痕迹等复杂 Word 元素可能不会被翻译，但通常会留在文档里。

## API Key 安全

如果勾选“记住 API Key”，工具会把 Key 明文保存在本文件夹的 `.translator_config.json`。如果这台电脑不是你个人使用，请不要勾选。

## 法律文件注意事项

本工具是翻译辅助工具，不构成法律意见。合同、协议、章程、授权书等法律文件在正式签署或提交前，应由专业律师或法律译审复核。

## 常见问题

### 拖入文件没反应

点击“选择文件”也可以使用。拖拽功能依赖 `tkinterdnd2`，个别电脑可能不可用。

### 提示没有 Python

如果是在这台电脑上运行，工具会优先使用 Codex 自带的 Python。只有把整个文件夹复制到另一台没有 Codex 运行环境的电脑时，才需要安装 Python 3.10 或更高版本；安装时勾选 `Add Python to PATH`，然后重新双击 `一键启动.bat`。

### 翻译很慢

如果使用逐段上下文模式，法律合同会逐段调用 API，并且要保留格式，所以比普通整篇翻译慢。想提速并保留粗体/下划线/高亮，请使用 `并发 run 保格式模式（推荐）`，并优先试 `deepseek-v4-flash`。

### DeepSeek 报错

先检查三件事：

1. API Key 是否是 DeepSeek 的 Key。
2. Base URL 是否是 `https://api.deepseek.com`。
3. 模型名称是否拼写正确：`deepseek-v4-flash`。

### 提示 WinError 5 / 拒绝访问

通常是输出文件正在被 Word、WPS、预览器或 OneDrive 占用。新版工具会尝试自动另存为一个带时间戳的新 `.docx`。如果仍然失败，请关闭已经打开的输出文档，或把输出位置改到桌面上的普通文件夹。

### 译文夹杂中文

新版默认使用并发 run 保格式模式，不会要求模型保留超大标签列表。若某个 run 漏翻，工具会只补译该 run，并把结果写回原 run，尽量避免中文残留，同时保留原 run 的粗体、下划线、高亮等格式。

### 输出文件打不开

请确认输出文件名以 `.docx` 结尾，并且没有把输出文件设置成原合同文件本身。
